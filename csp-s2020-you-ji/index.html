CSP-S2020游记

2020-11-13 17:55 8 min read # 游记

怎么就多出来一个比赛。

Day (-inf, 0]

考虑到这个比赛很可能很重要，就停了两周课。并且不是连着停，中间还去上了一周学。

停课期间基本上是在打 ZR 的模拟赛。发现自己很菜，也学到了一些东西。

最后一周基本上就在整理复习，花了很多时间在图论上，发现自己原先所谓的 ”学了“ 只是到 know about 的程度。于是边复习边跟着 lyd 的蓝书和 OI-Wiki 又 ”学了“ 一些东西，诸如最小生成树，欧拉序把 LCA 转 RMQ，和一些最短路算法等。

然后又打了一场 ZR 的比赛。

然后回顾了历年真题，不过有一些到现在都不太会打，也懒得看。

然后复习了数论。

然后看了看数据结构。考前一天打了个线段树区间加乘求和的板子，居然 WA 掉了，发现是乘法标记的有效条件是 tag!=1 而不是 tag>1 。

由于回顾的时候发现基本没考过字符串，所以就瞅了瞅 KMP 和 Manacher 敷衍一下求个心理安慰。

Day 0 晚上就开始坐立不安。

Day 1

为啥要把提高组放下午啊？早上都不知道怎么过。

于是开始看《球状闪电》。

看完了。

中午也不敢吃多，随便吃了点东西。说起来今年好像初中生没有几个去华科考的，我去 hsy。车上的时候我妈说普及组题目出来了，就看了两眼。“优秀的拆分”？“取数游戏”？出题人连新题名都想不出来了？还有 C 好像是 ZR 赛前连测出过的啊！然而我没打那场，只是看见了。到 hsy，一进校门就看见一个巨大的 華 。不仅字大，整个学校都大。

找到考场后首先去了趟厕所~~，也算摸清地形~~。

本来以为要抽签，结果发现座位已经投影出来了。扫了一眼，还是有一些认识的人的。然后发现 神 · FZ 座位在我斜对面。终于有机会一睹 神 的真容了吗......

接着就调了调编译器的设置，建好考号文件夹。看了看压缩包：julian 是个什么鬼名字？

开始了。

打开 PDF ，看概览，似乎一切正常。

往下翻，懵了。这个 A 是什么情况？本来我就不喜欢做和日期有关的题目，麻烦得要死，这还来个公元前，还教皇，还删什么日期，害人不浅啊！

再往下翻，B 看起来是个组合。

继续翻，这个 C 怎么和 ZR 连测的一题那么像啊？

翻，D 是关于蛇吃蛇的题，似乎和博弈有关系。那和贪吃蛇有个毛线关系？

回到 A 。

仔细思考了一下，显然可以数学推导 O(1)O(1)O(1) 回答询问。但我不太可能推出来。有没有暴力点的方法？算了一下好像年份跨度不会超过 O(109365)O(\dfrac{10^9}{365})O(365109​) (flag1) ，可以把询问离线了一个一个年份往后跳枚举，加一些复杂恶心的判断就可以了。于是开始打。一打就打了一个半小时。这是 A 啊！测一测样例，发现对于我的算法需要把输入加一，于是改了就没啥问题了。感觉复杂度也没啥问题。检查了一下 freopen 就扔了。

B 。随便画画发现对于每个 ppp ，只有已有的动物中有这一位为 000的情况下，才会有限制，要求选的动物这一为必须为 000 。其它没有限制的位则随便选。

那这不就做完了？准备用 map 处理一下输入，发现带 log⁡\loglog 可能炸，然后发现只需要把输入的 aaa 或起来就可以了。并且注意到 qqq 互不相同，做起来就容易多了。aaa 互不相同，最后答案就应该是 2k−t−n2^{k-t}-n2k−t−n ，其中 ttt 表示受限制的位置数。然后就意识到这题很恶心，不仅 kkk 到了 646464，还不给 ppp 的范围。不禁想起去年的 A 格雷码，坑死多少人。还好我敏锐意识到这一点，不会在奇怪的地方挂分 (flag2) 。并且注意到 t=0t=0t=0 时左移会出问题，所以直接用 unsigned long long 的自然溢出，换成 0−n0-n0−n 即可。特判也被我注意到了 (flag3) 。于是很快就解决掉了。

C 确实和那题很像，但是 ZR 那题是只会调用前面的函数，所以从后往前扫一遍即可。这个不仅随便调用，还可以调用多个。这时只剩不到两个小时，想了想转化成图，也没有很有前途的思路，就决定直接打暴力随缘了。于是就套了个区间加乘求和的线段树板子，正好是头一天打过的，所以很顺利，没出问题。然后就过了。

D 也没啥好想法，先把 n=3n=3n=3 的送分拿了，然后琢磨模拟这个过程，写了个 bool dfs(int x) 判断在当前局面下 xxx 有没有可能活下来，然后结合这个函数暴力模拟，估计能拿 20pts20pts20pts ，只要正确性没问题。

开始猜结论：只要吃了之后还有比自己小的就吃，否则结束。

感觉很有道理，于是打出来试试，结果和样例有 ±1\pm 1±1 的差距。果然没这么简单。

然后就快结束了。四个小时竟这样短。

再检查一下文件啥的，都没问题。

最后好像是用类似极域的东西手动交的文件夹。后来听说是为了节省时间，因为要各自确认自己代码无误，没有收上去的由工作人员下来收。

就结束了。

出来一点底都没有，大概是因为不知道怎么对拍，能做的题都没有很好的对拍方式，尤其是 A，只能随缘了。感觉比较可惜的是基本没考图论（除非说 C 和图有关系），而考前花了大把时间在图论，相对于其它板块更有信心。不过总会有用的。

走到 hsy 的广场上，感觉校园里已经一片漆黑。心情很复杂，总感觉真正考完这四个小时的感受与事先想的完全不同，题目不是恶心就是细节，难得又没挑战出来。抬起头看天，竟然隐约有几颗星星——果然地上没有亮灯。

晚上到 oitiku 上测了一下，发现自己 A 的最后一个点 T 了，再看题目：答案年份不超过 10910^9109 。俺看错了。

B 挂到了 60 多，打开程序一看，1<<p 格外引人注目，并且还要特判 k=64,n=0k=64,n=0k=64,n=0 的情况。为什么俺写格雷码时都能打出 1ull ，现在却栽在这种细节上？为什么特判会想不明白？错误总是出现在出乎意料的方面。

C 和 D 主要随缘就不管了。民间测出来大概 230pts230pts230pts 左右，如果 CCF 数据用脚造可能能上升一些。排名在 rank10rank 10rank10 左右，但这是 HB 。瞅瞅 ZJ 这种省，我这种弱鸡分数只能排到 rank100rank 100rank100 去了。

很烦，B 不应该挂分，但确实挂了。

A 不应该没看清数据范围，但确实看错了。

比较 Lucky 的是幸好 A 没看清最后一个点的规模，不然可能会想好久正解导致后面时间分配不均衡。

希望以后不再有不应该吧。
