公平组合游戏ICG

公平组合游戏 (Impartial Combinatorial Games) :

  • 两名选手交替进行游戏
  • 任一操作只与游戏本身有关，与操作选手等其他因素无关
  • 不存在平局

博弈论中，有一些有代表性的 ICG ，和一些常用方法。

Nim游戏

定义

通常的Nim游戏的定义是这样的：有若干堆石子，每堆石子的数量都是有限的，合法的移动是“选择一堆石子并拿走若干颗（不能不拿）”，如果轮到某个人时所有的石子堆都已经被拿空了，则判负（因为他此刻没有任何合法的移动）。

结论

记各堆石子数分别为 a1,a2,...,ana_1, a_2, ... , a_na1​,a2​,...,an​ 。
先手必胜，当且仅当 XOR⁡ ai≠0\operatorname{XOR} \, a_i \neq 0XORai​​=0

考虑证明。

当没有石子时，显然先手必败。

考虑 XOR⁡ ai=0\operatorname{XOR} \, a_i = 0XORai​=0 的局面。
此时一定不存在一种拿法，使得拿完之后其异或和仍为0。
假设存在，把 a1a_1a1​ 拿完变成 a1′a_1'a1′​。那么有 a1′XOR⁡a2XOR⁡...XOR⁡an=0a_1' \operatorname{XOR} a_2 \operatorname{XOR} ... \operatorname{XOR} a_n = 0a1′​XORa2​XOR...XORan​=0
则可知 a1=a1′a_1 = a_1'a1​=a1′​，相当于没有拿，这是不合法的。
故一定不存在这种拿法。
故先手会一直因后手的操作，保持在异或和为 000 的局面，直至最后没有石子。
故先手必败。

换言之，先手必胜，当且仅当 XOR⁡ ai≠0\operatorname{XOR} \, a_i \neq 0XORai​​=0。

有向图游戏

给定一个 有向无环图 和一个起始顶点上的一枚棋子，两名选手交替的将这枚棋子沿有向边进行移动，无法移 动者判负。事实上，这个游戏可以认为是 所有公平组合游戏的抽象模型。

解决这种问题，常用 SG 函数

SG函数

定义

记 g(x)g(x)g(x) 为点 xxx 的 SG 函数。引入 mex⁡\operatorname{mex}mex 运算 ，mex⁡{a1,a2,...,an}\operatorname{mex}\{ a_1, a_2, ... , a_n\}mex{a1​,a2​,...,an​} 的结果为此集合中未出现的最小自然数。

那么 g(x)=mex⁡{g(ui)}g(x) = \operatorname{mex} \{g(u_i)\}g(x)=mex{g(ui​)} ，其中 uiu_iui​ 为 xxx 所指向的节点。

结论

对于上述有向图游戏，记起点为 sss ，那么 先手必胜当且仅当 g(s)≠0g(s) \neq 0g(s)​=0 。根据每步操作带来的变化，可以用类似于 Nim游戏 的证明方法加以证明。

对于多个有无环向图，则先手必胜当且仅当 XOR⁡g(si)≠0\operatorname{XOR} g(s_i) \neq 0XORg(si​)​=0 。

求法

对于一个有向无环图，dfs 即可计算求解 SG 函数。

acoj - 12754 - 移棋子游戏

/**********************************

Problem: acoj - 12754 - 移棋子游戏

Author: cyh_toby

#include <cstdio>
#include <cstring>
#include <set>

using namespace std;

const int N = 2005, M = 6005;

int g[N], nxt[M], v[M], idx;
int n, m, k;
int f[N];//SG
int ans = 0;

inline void add(int x, int y) {
	idx++;
	v[idx] = y;
	nxt[idx] = g[x];
	g[x] = idx;
	return;
}

int dfs(int u) {//求u的SG值 
	if (f[u] >= 0)
		return f[u];
	
	f[u] = 0;
	set<int> s;
	for (int i = g[u]; i; i = nxt[i])
		s.insert(dfs(v[i]));
	
	for (int i = 0; ; i++) {
		if (!s.count(i)) {
			f[u] = i;
			break;
		}
	}
	
	return f[u];
}

int main()
{
	scanf("%d%d%d", &n, &m, &k);
	for (int i = 1; i <= m; i++) {
		int a, b;
		scanf("%d%d", &a, &b);
		add(a, b);
	}
	
	memset(f, -1, sizeof(f));
	
	for (int i = 1; i <= k; i++) {
		int x;
		scanf("%d", &x);
		ans ^= dfs(x);
	}
	
	if (ans)
		puts("win");
	else
		puts("lose");
	
	return 0;
}

acoj12755牛的数字游戏

acoj - 12755 - 牛的数字游戏

把每个数字拆分开，按题意模拟建图，使用 SG 函数。

/**********************************

Problem: acoj - 12755 - 牛的数字游戏 

State: Accepted

From: acoj

Algorithm: 博弈论, SG函数 

Author: cyh_toby

Last updated on 2020.5.31

**********************************/
#include <cstdio>
#include <algorithm>

using namespace std;

const int N = 1000005;

int f[N];

inline void getm(int x, int &r1, int &r2) {
	r1 = 0, r2 = 10;
	while (x) {
		r1 = max(r1, x % 10);
		r2 = min(r2, x % 10 == 0 ? 10 : (x % 10));
		x /= 10;
	}
	return;
}

inline void init() {
	f[0] = 0;
	for (int i = 1; i <= N-5; i++) {
		int x, y;
		getm(i, x, y);
		x = i - x, y = i - y;
		x = f[x], y = f[y];
		if (x < y)
			swap(x, y);
			
		if ((x == 0 && y == 0) || (x == 2 && y == 0))
			f[i] = 1;
		if (y > 0)
			f[i] = 0;
		if (x == 1 && y == 0)
			f[i] = 2;
	}
	return;
}

int main()
{
	init();
	
	int T;
	scanf("%d", &T);
	while (T--) {
		int n;
		scanf("%d", &n);
		if (f[n])
			puts("YES");
		else
			puts("NO");
	}
	return 0;
}

acoj12756取石子游戏

acoj - 12756 - 取石子游戏

与上一题类似，根据取石子的限制条件建图（可以不用真的建出来），随后 SG 求解。

/**********************************

Problem: acoj - 12756 - 取石子游戏 

State: Accepted

From: acoj

Algorithm: 博弈论, SG函数 

Author: cyh_toby

Last updated on 2020.5.31

**********************************/
#include <cstdio>
#include <set>
#include <cstring>

using namespace std;
const int N = 15, K = 1005;

int n, m, a[N], b[N], f[K];
int book[K];

inline void init() {
	f[0] = 0;
	for (int i = 1; i < K; i++) {
		memset(book, 0, sizeof(book));
		for (int j = 1; j <= m; j++) {
			if (i < b[j]) break;
			book[f[i-b[j]]] = 1;
		}
		for (int j = 0; ; j++) {
			if (!book[j]) {
				f[i] = j;
				break;
			}
		}
	}
	return;
}

int main()
{
	scanf("%d", &n);
	for (int i = 1; i <= n; i++)
		scanf("%d", &a[i]);
	scanf("%d", &m);
	for (int i = 1; i <= m; i++)
		scanf("%d", &b[i]);
	
	init();
	
	int res = 0;
	for (int i = 1; i <= n; i++)
		res ^= f[a[i]];
	
	if (res)
		puts("YES");
	else
		puts("NO");
	
	for (int i = 1; i <= n; i++) {
		for (int j = 1; j <= m; j++) {
			if (a[i] - b[j] >= 0 && ((res ^ f[a[i]] ^ f[a[i] - b[j]]) == 0)) {
				printf("%d %d\n", i, b[j]);
				return 0;
			}
		}
	}
	
	return 0;
}

SPOJ9934/acoj12757取石子

SPOJ9934 / acoj12757 - 取石子

分析

这道题有两种操作，合并 或 取一 。显然两种操作的无法进行的时候，游戏结束，当前玩家负。

那么我们可以从操作总数入手分析。

首先，我们容易发现，对于一堆石子，需要分情况讨论石子数大于等于 222 和 等于 111 的局面。因为

  • 大于等于 222 的时候，无论取一还是合并，都只会对操作总数减一
  • 等于 111 的时候，合并会对操作数减一，但取一操作会使操作数减二 （因为少了一个石子操作，同时少了一个堆可以操作）

那么，就把这两种情况分开。具体来说，用 f\(a, b\) 表示某种情况是否先手必胜，其中

  • aaa 表示，所有只有一个石子的石堆中，石堆数。
  • bbb 表示，所有大于等于二的个石子的石堆中，石堆数 +++ 石子数 −-− 111

现在单独分析 bbb。

首先给出结论，bbb 为奇数先手必胜。

当只有一堆石子的时候，b = 1 +b\,=\,1\,+b=1+ 石子数 − 1=-\,1 =−1= 石子数，那么显然 bbb 为奇数则先手必胜。

在有多堆石子的时候，（务必时刻记住，bbb 所表示的是 大于等于二个石子 的石堆中的统计）

  • 若 bbb 为奇数，那么操作完 bbb 必定变为偶数
  • 若 bbb 为偶数
    • 假设对石子数大于 222 的石堆操作，那么不论怎么操作，bbb 都会变成奇数
    • 假设有一堆石子数为 222 并且对这一堆进行操作，如果是合并，将同于上钟情况；如果是取一，则接下来这一堆变成石子数为 111 的石堆。现在 bbb 变成奇数，而这时若先手选择取走，则 bbb 仍保持奇数，那么原本的后手将会有机会翻转局面，必胜；所以必须选择合并（这种情况有多堆石子，可以合并），才能维持后手拿到的 bbb 是偶数。

以上是对 bbb 的分析，发现上述结论，即“ bbb 为奇数先手必胜”，是正确的。接下来开始分析 f(a,b)f\big(a,b\big)f(a,b) 。

为了便于表示，把只有一个石子的石堆放入集合 xxx 中，其他放入 yyy 中。

xyx\qquad\qquad\qquad\qquad\qquad\qquad y xy

则 f(a,b)f(a,b)f(a,b) 的后继情况为

  • 从 xxx 中拿一个石子，则变为 f(a−1,b)f(a-1, b)f(a−1,b)
  • 从 yyy 中拿一个石子，则变为 f(a,b−1)f(a, b-1)f(a,b−1)
  • 合并 xxx 中两堆，则变为 f(a−2,b+3)f(a-2, b+3)f(a−2,b+3)
  • 合并 yyy 中两堆，则变为 f(a,b−1)f(a, b-1)f(a,b−1)
  • 合并 xxx 和 yyy 中两堆，则变为 f(a−1,b+1)f(a-1, b+1)f(a−1,b+1)
    这其中任意一个情况必败，则 f(a,b)f(a, b)f(a,b) 必胜。否则必败。

因此，在实现的时候只需要用 f(a,b)f(a, b)f(a,b) 存 111 或 000 即可。

实现细节

需要注意的是，上面 555 种情况分别有各自条件，需要先判断是否可行。

此外，根据 bbb 的定义，在 b=0b=0b=0 和 b≠0b \ne 0b​=0 时，对其进行增加是需要分类处理的，具体见代码。

此外，我们发现在进行第二种操作的时候，yyy 中可能出现某一堆石子只含一个石子的情况。根据上文对 bbb 的分析，这个时候如果还有多堆，必须选择合并操作，这样就不会继续保持 yyy 中的不符合定义局面，并且也必须这样做才能保持当前手必胜。而当 b=1b=1b=1 时，说明只有一堆且这一堆石子数为一，那么特殊判断把情况更改为 f(a+1,0)f(a+1, 0)f(a+1,0) 即可。

我们发现 222 和 444 的后继状态时一样的，而由于转移 222 状态的潜在条件是 必须有多堆（这一点正如上一段所说） ，因此 222 合 444 的可行条件是一样的，可以归为一种（指在写代码的时候）。

/**********************************

Problem: SPOJ9934 / acoj - 12757 - 取石子

#include <cstdio>
#include <cstring>

using namespace std;

const int N = 55;
int n, f[55][50055];

int dfs(int a, int b) {
	if (f[a][b] >= 0) 
		return f[a][b];
	if (a == 0)
		return f[a][b] = b % 2;
	if (b == 1)
		return dfs(a+1, 0);
		
//  -----	
	
	if (a && dfs(a-1, b) == 0)
		return f[a][b] = 1;
		
	if (b && dfs(a, b-1) == 0)
		return f[a][b] = 1;
		
	if (a >= 2 && dfs(a-2, b==0? b+2 : b+3) == 0)
		return f[a][b] = 1;
		
	if (a && b && dfs(a-1, b+1) == 0)
		return f[a][b] = 1;
	
	return f[a][b] = 0;
	
}

int main() 
{
	memset(f, -1, sizeof(f));
	int T;
	scanf("%d", &T);
	while (T--) {
		scanf("%d", &n);
		int a = 0, b = 0;
		for (int i = 1; i <= n; i++) {
			int x;
			scanf("%d", &x);
			if (x == 1)
				a++;
			else {
				if (b == 0)
					b += x;
				else
					b += x+1;
			}
		}
		
		if (dfs(a, b))
			puts("YES");
		else
			puts("NO");
	}
	
	return 0;
}
