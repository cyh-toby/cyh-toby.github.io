学习笔记：图论-LCA

2020-01-17 16:57 6 min read # 学习笔记

概念

对于有根树TTT的两个结点uuu、vvv，最近公共祖先LCA(T,u,v)LCA(T,u,v)LCA(T,u,v)表示一个结点xxx，满足xxx是uuu和vvv的祖先且xxx的深度尽可能大。在这里，一个节点也可以是它自己的祖先。

分析一：离线-tarjantarjantarjan

tarjantarjantarjan可以在O(N+Q)O(N+Q)O(N+Q)的时间复杂度内离线求解LCALCALCA问题，其实现依赖于并查集。

  • 起初，每个点的并查集源是自己；在结束对一个点和其子树的遍历后，将其与其父亲合并。

  • 搜索完一个点的所有子树后，检查与此点相关的询问，假设当前点为uuu，其询问的点为vvv，若发现vvv已经被搜索过(vis[v]==1vis[v]==1vis[v]==1)，那么LCA(u,v)=f[v]LCA(u,v) = f[v]LCA(u,v)=f[v]//此处f[v]f[v]f[v]表示vvv的并查集源。

分析二：在线-倍增

倍增算法需要O(NlogN)O(NlogN)O(NlogN)的预处理，存下每个点的祖先信息，然后以每次O(logN)O(logN)O(logN)的时间复杂度查询。

  • 预处理step1step1step1：声明数组anc[N][log2N]anc[N][log_2N]anc[N][log2​N]，其中，anc[i][j]anc[i][j]anc[i][j]表示iii号节点的第2j2^j2j个祖先编号（与STSTST表类似）。

  • 预处理step2step2step2：我们用dfsdfsdfs遍历全图，同时记录深度。对于∀u∈G∀u∈G∀u∈G，anc[u][0]=father[u]anc[u][0]=father[u]anc[u][0]=father[u]，然后从当前点向上处理，即

		anc[u][i] = anc[anc[u][i-1]][i-1]
  • 查询step1step1step1：假设询问的点为xxx和yyy。首先，我们令xxx成为深度较大的那个点(便于操作)，然后使xxx不停向上跳，直到与yyy深度相同。这时，倍增的优势就显现出来了：一层一层跳当然不会错，但是效率低，所以采用一次跳2i2^i2i格，并且iii从大到小，每次都尽可能多地跳，这样就可以刚好找到并且复杂度由O(N)O(N)O(N)降为O(logN)O(logN)O(logN)了。

  • 注意: 此处一定要记得特判，这种情况就相当于yyy就是xxx的祖先；如果不特判，继续如下操作则会出错。

  • 查询step2step2step2：现在xxx和yyy已经在同一层，那么显然我们要把他俩一起向上跳，直到刚好重合（不多也不能少）。一起跳很容易实现，但怎么实现刚好重合呢？如果我们写if(anc[x][i]==anc[y][i])if(anc[x][i] == anc[y][i])if(anc[x][i]==anc[y][i])就往上跳，那很有可能会跳多，就是说跳到了LCALCALCA的上方。所以，我们应该把======改成!=!=!=，才能保证一直跳在LCALCALCA下方。这样运行结束后，LCALCALCA就刚好是xxx和yyy的父亲了。

            x = anc[x][i];

    }
    return anc[x][0];

利用好倍增，查询就是O(logN)O(logN)O(logN)的时间复杂度了。

代码

//倍增+前向星 
#include <cstdio>
#include <algorithm>
#include <cmath>
#include <cstring>
using namespace std;
#define N 500005
#define gc getchar
int n, m, rt;
int anc[N][20];
int dep[N];
struct Node{
	int v;
	int nxt;
} node[2*N];
int graph[N];
int tot = 0; 
inline int rd() {
	int x = 0, f = 1;
	char c = gc();
	while (c<'0'||c>'9') {
		if(c=='-') f = -1;
		c = gc();
	} 
	while (c>='0'&&c<='9') {
		x = (x<<1) + (x<<3) + c - '0';
		c = gc();
	} 
	return x*f;
}
inline void add(int x, int y) {
	node[++tot].v = y;
	node[tot].nxt = graph[x];
	graph[x] = tot;
	node[++tot].v = x;
	node[tot].nxt = graph[y];
	graph[y] = tot;
	return;
}
void dfs(int cur, int father, int depth) {//深度从0开始计，那么某点祖先数就是它的深度 
	dep[cur] = depth;
	anc[cur][0] = father;
	for (int i = 1; (1<<i) <= depth ; i++) {
		anc[cur][i] = anc[anc[cur][i-1]][i-1];
	}
	for (int i = graph[cur]; i; i = node[i].nxt) {
		int t =	node[i].v;
		if (t==father) continue;
		dfs(t, cur, depth+1);
	}
	return;
}
inline int ask(int x, int y) {
	if (dep[x]<dep[y]) swap(x, y);
	int derta = dep[x]-dep[y];
	for (int i = 0; (1<<i)<=derta; i++) {
		if ((1<<i) & derta) {//temp第i位是1，需要向上跳 
			x = anc[x][i];
		}
	}	
	if (x==y) return x;
	int p = log2((double)dep[x]);
	for (int i = p; i >= 0; i--) {
		if (anc[x][i]!=anc[y][i]) x = anc[x][i], y = anc[y][i];
	}
	return anc[x][0];
}
int main() {
	n = rd(), m = rd(), rt = rd();
	for (int i = 1; i <= n-1; i++) {
		int u = rd(), v = rd();
		add(u, v);
	}
	dfs(rt, 0, 0);
	for (int i = 1; i <= m; i++) {
		int u = rd(), v = rd();
		printf("%d\n", ask(u, v));
	}
	return 0;
}

小结

个人认为倍增会比tarjantarjantarjan更实用。

其实倍增重要的并不是算法，而是倍增思路和技巧，切忌死搬硬套。掌握了倍增思想可以更有助于破题。
