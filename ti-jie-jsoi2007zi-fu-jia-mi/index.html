题解 【[JSOI2007]字符加密】

2020-07-25 21:23 4 min read # 字符串 # SA # 题解

题意

原题：[JSOI2007]字符加密

分析

显然先要断环成链。把给定字符串复制一遍放在后面。

然后对其求一遍后缀数组，发现就相当于对题目描述的那些东西排了序（当然最后需要把长度过短的忽略）。

为什么？

假设现比较后缀 iii 与后缀 jjj 的大小关系。

  • 如果 s[i:i+n−1]>s[j:j+n−1]s[i:i+n-1] > s[j:j+n-1]s[i:i+n−1]>s[j:j+n−1]，那不管给它们后面添上什么东西都不会改变其大小关系，因此后缀 iii 必然大于后缀 jjj 。反之类似。
  • 如果 s[i:i+n−1]=s[j:j+n−1]s[i:i+n-1] = s[j:j+n-1]s[i:i+n−1]=s[j:j+n−1]，说明两个串相同，那么它们谁先谁后都一样了，所以由它们后面的字符决定其先后大小关系也无所谓。

所以，影响一个后缀 s[i:2n]s[i:2n]s[i:2n] 与其他后缀大小关系的，其实就是 s[i:i+n−1]s[i:i+n-1]s[i:i+n−1] 。

反过来，后缀 i  ≥i\;\gei≥ 后缀 jjj 必然意味着 s[i:i+n−1]≥s[j:j+n−1]s[i:i+n-1] \ge s[j:j+n-1]s[i:i+n−1]≥s[j:j+n−1] 。

所以求后缀数组即可。

之后有一些细节操作，见代码。

源码

const int N = 2e5+5;

int n, w;
char s[N];
int sa[N], rk[N<<1], oldrk[N<<1], cnt[N], id[N];

inline bool cmp(int x, int y, int j) {
	return oldrk[x] == oldrk[y] && oldrk[x+j] == oldrk[y+j];
}

int main()
{
	scanf("%s", s+1);
	n = strlen(s+1);
	for (int i = 1; i <= n; i++) 
		s[i+n] = s[i];
	n <<= 1;
	
	for (int i = 1; i <= n; i++) cnt[(int)s[i]]++;
	w = 300;
	for (int i = 1; i <= w; i++) cnt[i] += cnt[i-1];
	for (int i = n; i >= 1; i--) sa[cnt[(int)s[i]]--] = i;
	w = 0;
	for (int i = 1; i <= n; i++)
		rk[sa[i]] = s[sa[i]] == s[sa[i-1]] ? w : ++w;
	
	for (int j = 1; j < n; j <<= 1) {
		int t = 0;
		for (int i = n; i > n-j; i--) id[++t] = i;
		for (int i = 1; i <= n; i++)
			if (sa[i] > j) id[++t] = sa[i] - j;
		
		memset(cnt, 0, sizeof(cnt));
		for (int i = 1; i <= n; i++) cnt[rk[id[i]]]++;
		for (int i = 1; i <= w; i++) cnt[i] += cnt[i-1];
		for (int i = n; i >= 1; i--) sa[cnt[rk[id[i]]]--] = id[i];
		
		memcpy(oldrk, rk, sizeof(oldrk));
		w = 0;
		for (int i = 1; i <= n; i++) 
			rk[sa[i]] = cmp(sa[i], sa[i-1], j) ? w : ++w;
	}
	
	for (int i = 1; i <= n; i++) 
		if (sa[i] <= n/2) printf("%c", s[sa[i]+n/2-1]);
	
	return 0;
}
