T1 【序列】

题意

原题：P6185 [NOI Online 提高组]序列

给定初始序列 aaa 与目标序列 bbb ，并且给定一些形如下述的操作，判断是否可以通过这些操作（顺序次数不限）把初始序列变成目标序列。

  • 操作1    i ,  jai+x ,  aj+x ,  x∈{1,−1}1\;\;i\,,\;j\qquad a_i + x\,,\;a_j + x\,,\;x\in\{1,-1\}1i,jai​+x,aj​+x,x∈{1,−1}

  • 操作2    i ,  jai+x ,  aj−x ,  x∈Z2\;\;i\,,\;j\qquad a_i + x\,,\;a_j - x\,,\;x\in Z2i,jai​+x,aj​−x,x∈Z

分析

首先令 bi=bi−aib_i=b_i-a_ibi​=bi​−ai​ , 那么所有 aia_iai​ 就都变成 000 了，便于写代码。以下分析皆基于进行此操作后的序列。即一下分析中，视每个的初始值为 000 。

显然，没有什么比较好的方法能解决类似于此的关联性大的复杂操作，除了图。考虑转化图结构。

操作 2

首先考虑操作 222 ，其实就是两个元素之间可以把权相互运输/传递。那么对于每个操作 222 ，建立一条双向边，形成图 GGG , 则 GGG 中同一通块中的点之间可以互相传递权。严谨一点，就是一个连通块可以在总和不变的情况下使其中的点任意加减。所以只需满足连通块的 ∑bi=0\sum b_i=0∑bi​=0 即有解。

为了便于操作，把这种连通块缩点，构建新图 G′G'G′

对于原节点 xxx ，其所在连通块记为 idxid_xidx​

下文提到的图皆指新图 G′G'G′

操作 1

考虑操作 111 ，发现这个操作可以使总和发生变化。那么不太好直接操作。

尝试与操作 222 找到某种共通之处。

首先根据所有操作 111 给新图加边。即1  x  y  ⟺  (idx,idy)1\;x\;y \iff (id_x,id_y)1xy⟺(idx​,idy​) 。

如果 idx=idyid_x = id_yidx​=idy​ ,也就是自环，那就是这个点可以自增 2k(k∈Z)2k\quad \big(k\in Z\big)2k(k∈Z) ,为其打上标记 tag=1tag=1tag=1 ，但不实际加边。对于 tag=1tag=1tag=1 的点，有解条件扩增为 ∑bi\sum b_i∑bi​ 为偶数

发现，对于两点 x , yx\,,\, yx,y ，如果存在一条路径(不一定是简单路径)长度为偶数，则 x , yx\,,\,yx,y 之间可以进行类似操作 222 的权值传递。证明如图：

长度大于 222 可归纳法证明。

因此考虑找到所有这样的点，可以再次像操作 222 那样缩点，判断条件也一样。

而我们知道，一个包含奇数个点的环上任意两点之间，存在一条长度为奇的路径和长度为偶的路径。这意味着，奇环可以作为一个转化器，只要一条路径经过奇环，长度就可以在奇偶间任意选择。也就是说，如果一个连通块包含奇环，则这个连通块内所有点之间可以相互转运权值。也相当于操作 222 ，但由于这些点(视此情况默认点数大于 111)是由操作 111 的边连通的，故缩成的新点有自环，即 tag=1tag=1tag=1 ，所以有解条件是连通块 ∑bi\sum b_i∑bi​ 为偶数。

那么如何判断一个连通块是否包含奇环呢？易发现 没有奇环   \; 是   \; 图为二分图 的 充要条件。

所以判断一个连通块是不是二分图即可。

  • 不是二分图。那么有解条件就是 ∑bi\sum b_i∑bi​ 为偶数。

  • 是二分图。为其左右染色。由于是二分图且连通，所以同色节点之间两两存在长度为偶数的路径，即可以在总和不变的情况下任意加减，又可以缩点。因此把二分图想象成两个点。则这两个点可以在差值不变的情况下任意加减，所以有解条件为两个条件同时满足

    1. ∑左bi\sum\limits_\text{左} b_i左∑​bi​ 与 ∑右bi\sum\limits_\text{右} b_i右∑​bi​ 奇偶相同，即 ∑左和右bi\sum\limits_\text{左和右} b_i左和右∑​bi​ 为偶数。

    2. 如果所有 tag=0tag=0tag=0 ，即两边无法分别自增 2k2k2k，则能在保证两边差值不变的情况下整体自增，所以需要 ∑左bi=∑左bi\sum\limits_\text{左} b_i = \sum\limits_\text{左} b_i左∑​bi​=左∑​bi​ (再次强调此处是指经过开篇所述处理之后的 b)

当所有点都有解，整体就有解，否则无解。

第二次即操作 111 中无需真的缩点，只需要判定二分图的时候顺带处理统计一下即可。

/**********************************

Problem: [NOIO2020提高组] - 序列 

State: Accepted 

From: [NOIO2020提高组]

Algorithm: 图论, 转化图, 二分图判定, 

Author: cyh_toby

#include <cstdio>
#include <vector>
#include <cstring>
using namespace std;
#define ll long long
#define pb push_back
const int N = 100005;
int n, m;
ll b[N];
struct Operation{
	int x, y;
} op[N];
int cnt, num;
vector<int> g1[N], g2[N];
int id[N];
ll w[N];//新图点权 
int tag[N], vis[N];
bool ans;
inline void resets() {
	cnt = num = 0;
	ans = true;
	memset(id, 0, sizeof(id));
	memset(w, 0, sizeof(w));
	memset(tag, 0, sizeof(tag));
	memset(vis, -1, sizeof(vis));
	for (int i = 1; i < N; i++)
		g1[i].clear(), g2[i].clear();
	return;
}
void dfs1(int u, int c) {
	id[u] = c;
	w[c] += b[u];
	int len = g1[u].size();
	for (int i = 0; i < len; i++) {
		int t = g1[u][i];
		if (id[t]) continue;//来的点
		dfs1(t, c); 
	}
	return;
}
inline void add(int x, int y) {//此处x,y已经是id 
	if (x == y)
		tag[x] = 1;//可以自增 2k 
	else 
		g2[x].pb(y), g2[y].pb(x);
	return;
}
bool dfs2(int u, int c, int &tg, ll &s0, ll &s1) {
	vis[u] = c;
	if (c == 1)//分别统计二分图两侧的点的 bi-ai 之和, 免得判定完之后再 dfs 
		s1 += w[u];
	else
		s0 += w[u];
	tg |= tag[u];
	int len = g2[u].size();
	for (int i = 0; i < len; i++) {
		int t = g2[u][i];
		if (vis[t]==-1)
			dfs2(t, c^1, tg, s0, s1);
		else if (vis[t] == c)
			return false;		
	}
	return true;
}
int main() {
	int T;
	scanf("%d", &T);
	while (T--) {
		resets();
		scanf("%d%d", &n, &m);
		for (int i = 1; i <= n; i++)
			scanf("%lld", &b[i]);
		for (int i = 1; i <= n; i++) {
			ll x;
			scanf("%lld", &x);
			b[i] = x - b[i];
		}
		for (int i = 1; i <= m; i++) {
			int ope, x, y;
			scanf("%d%d%d", &ope, &x, &y);
			if (ope == 1) 
				op[++cnt].x = x, op[cnt].y = y;
			else
				g1[x].pb(y), g1[y].pb(x);
		}
		for (int i = 1; i <= n; i++)
			if (!id[i])
				dfs1(i, ++num);
		for (int i = 1; i <= cnt; i++) {
			int x = op[i].x, y = op[i].y;
			add(id[x], id[y]);
		}
		for (int i = 1; i <= num; i++) {//判定二分图 
			if (vis[i] != -1) continue;
			ll s0 = 0, s1 = 0;
			int tg = 0;
			bool flag = dfs2(i, 0, tg, s0, s1);
			if (flag) {//此连通块是二分图 
				if ((s0&1) != (s1&1)) ans = false;//不同奇偶，即使tag=1可以自增，也无法改变奇偶，故无解 
				if (!tg && s0 != s1) ans = false;//任意一边没法自增，只能两边同时加减，而b左-a左 ！= b右-a右，无解 
			}
			else //不是二分图，一定存在奇环，则两点之间一定有偶路径，则任意两点可以互相运输 
				if ((s0+s1) % 2 != 0)// 由于是个点数大于1的连通块，必定是op=1的边将其连接，即连通块可以自增2k，而bi - ai的和不是偶数，无解 
					ans = false;
		}
		if (ans) printf("YES\n");
		else printf("NO\n");
	}
	return 0;
}

T2 【冒泡排序】

题意

原题：P6186 [NOI Online 提高组]冒泡排序（民间数据）

给定一个序列，有操作和询问

  • 操作，交换相邻两个数

  • 询问，若经过 kkk 轮冒泡排序后的逆序对个数

分析

涉及到冒泡排序基本上都和逆序对有关，即归并排序或树状数组。

记 cic_ici​ 为序列 [1 , n][1\,,\,n][1,n] 子序列中的逆序对个数。

注意到一个很有用的性质，如下

经过一轮冒泡排序，新的逆序对个数数组 ci′=max⁡(0 , ci+1−1)c_i' = \max(0\,,\, c_{i+1}-1)ci′​=max(0,ci+1​−1)

考虑证明。参考于 xht37 的证明，末尾处有链接。

一轮冒泡，就相当于把 iii 位置的数挪到第一个数值大于它的位置 jjj 的前一位，而中间的数全部向左挪一位。此后，再对 jjj 进行类似操作，依次类推。

那么显然中间位置的数都会 ck−1c_k-1ck​−1 。而这样的 iii 的逆序对个数 cic_ici​ 一定为 000 （否则就不会挪动它），同理 cjc_jcj​ 也一定为 000 ，因而 cj−1′=max⁡(cj−1 , 0)=max⁡(−1,0)=0c_{j-1}'=\max(c_j-1\,,\,0) = \max(-1,0) = 0cj−1′​=max(cj​−1,0)=max(−1,0)=0 成立。

所以此性质正确。

这意味着，每轮冒泡排序，会使所有大于 000 的 cic_ici​ 通通减一，则总逆序对个数减去大于 000 的 cic_ici​ 的个数。由于只关心总和，我们不需要细究到底是哪几个 cic_ici​ 减少了 111 ，只需要统计其数量。

因此，易知， kkk 轮冒泡后的逆序对个数为

∑ci>k(ci−k)\sum\limits_{c_i>k}\big(c_i-k\big) ci​>k∑​(ci​−k)

但 kkk 是多次询问的不定量，而 cic_ici​ 是定量，因此为了更方便地维护，我们把上式展开，即

(∑ci>kci)−k×∑ci>k1\Bigg(\sum\limits_{c_i>k}c_i\Bigg)-k\times\sum\limits_{c_i>k}1 (ci​>k∑​ci​)−k×ci​>k∑​1

即需要求 ci>kc_i>kci​>k 的所有 cic_ici​ 之和以及个数。

因此，使用以 cic_ici​ 为关键字的权值树状数组维护即可。而这个树状数组需要维护两个量，即上式中的两个 ∑\sum∑ 。

对于交换操作，记 $x=a_i,,,y=a_{i+1} $ ，那么有两种情况

  • x>yx > yx>y , 这样交换后，yyy 对应的 ccc 值减一， xxx 的不变。

  • x<yx < yx<y ，这样交换后，xxx 对应的 ccc 值加一，yyy 的不变。

而对于 ccc 值的变化，就相当于删去一个原 ccc ，新加入一个 c′c'c′ ，即先提供一个负权，再提供一个正权，一样可以用权值树状数组维护。详见代码。

总体理一下，就是先用树状数组或者归并排序求出 ccc 数组，再以此建立权值树状数组进行维护即可。

显然 ∀i  ci<n\forall i\,\,c_i<n∀ici​<n，所以权值树状数组建树不用离散化，只需要给统计答案的容器开个 long long 即可。此外还要注意特判 k>nk>nk>n 的情况避免访问越界。

/**********************************

#include <cstdio>
#include <utility>
#include <iostream>//swap
using namespace std;
#define ll long long
#define pli pair<long long,int>
#define mp make_pair
const int N = 200005;
int n, m;
int a[N], t[N], c[N];//原数组，原数组权值树状数组，逆序对个数
pli d[N];//逆序对个数的权值树状数组 
inline void updat1(int x, int v) {
	for (; x <= n; x += x & (-x))
		t[x] += v;
	return;
} 
inline int query1(int x) {
	int res = 0;
	for (; x; x -= x & (-x))
		res += t[x];
	return res;
}
inline void updat2(int x, ll w, int v) {
	if (x == 0) 
		return;
	for (; x <= n; x += x & (-x))
		d[x].first += w, d[x].second += v;
	return;
}
inline pli query2(int x) {
	pli res;
	res.first = 0ll, res.second = 0;
	for (; x; x -= x & (-x))
		res.first += d[x].first, res.second += d[x].second;
	return res;
}
inline void cg(int x) {//交换操作,x和x+1 
	if (a[x] > a[x+1]) {
		updat2(c[x+1], -1ll*c[x+1], -1);
		c[x+1]--;
		updat2(c[x+1], 1ll*c[x+1], 1);
	}
	else {
		updat2(c[x], -1ll*c[x], -1);
		c[x]++;
		updat2(c[x], 1ll*c[x], 1);
	}
	swap(a[x], a[x+1]);
	swap(c[x], c[x+1]);
	return;
}
inline void qy(int k) {
	if (k > n) {//特判，避免访问越界 
		printf("0\n");
		return;
	}
	pli x = query2(n), y = query2(k);
	printf("%lld\n", x.first-y.first-1ll*k*(x.second-y.second));
	return;
}
int main()
{
	scanf("%d%d", &n, &m);
	for (int i = 1; i <= n; i++) {
		scanf("%d", &a[i]);
		c[i] = query1(n) - query1(a[i]);
		updat1(a[i],1);
	}
	for (int i = 1; i <= n; i++) 
		updat2(c[i], c[i], 1);
	for (int i = 1; i <= m; i++) {
		int op, x;
		scanf("%d%d", &op, &x);
		if (op == 1) 
			cg(x);
		else
			qy(x);	
	}
	return 0;
}

T3 【最小环】

题意

原题：P6187 [NOI Online 提高组]最小环

  • 给一个环，多次询问。

  • 定义环上 i , j  (i≤j)i\,,\,j\;(i\leq j)i,j(i≤j) 的距离指 min⁡(j−i ,  i+n−j)\min(j-i\,,\;i+n-j)min(j−i,i+n−j)

经过某种排序把相距 kkk 的点分别找出来，重新按某种顺序凑成若干个小环，并求所有环上的相邻数的乘积的和。也就是要达到如下方图 111 的效果

有多少个小环？

显然每个小环的点数是一样的。设小环上点数为 xxx ，小环个数为 yyy 。

显然，在图 111 中，x×k=nx \times k = nx×k=n，因为 kkk 刚好是 nnn 的约数。但在图 222 （即n ⁣= ⁣5 , k ⁣= ⁣2n\!=\!5\,,\,k\!=\!2n=5,k=2）中不是这样的，整个大环其实被分配到同一个小环中， 但“绕”了不止一圈。稍加思考发现， x×kx\times kx×k 总是 nnn 的倍数。可设

x×k=r×n(r∈N+)  x\times k = r\times n\quad \big(r \in N_+\big)\; x×k=r×n(r∈N+​)

则 y=nx=kry=\dfrac{n}{x}=\dfrac{k}{r}y=xn​=rk​ 。

而注意到 x=r×nkx=\dfrac{r\times n}{k}x=kr×n​ 为正整数，并且应找到最小的(因为回到出发点就不可以继续绕环了，故要找最小的) r∈N+r\in N_+r∈N+​ 使其成立，那么 rrr 应该满足 “ 补齐了nnn 中没有但 kkk 中有的因子 ”，即

r=kgcd⁡(n,k)r = \dfrac{k}{\gcd(n, k)} r=gcd(n,k)k​

所以知

y=gcd⁡(n,k)  ,  x=ngcd⁡(n,k)y=\gcd(n,k)\;,\;x=\dfrac{n}{\gcd(n,k)} y=gcd(n,k),x=gcd(n,k)n​

也就是，环的个数是确定且已知的。

下一步，找到合理分配方案，显然贪心

直觉结论：大的和大的放在同一个环，并且大的和大的乘。

/**********************************

Problem: [NOIO2020提高组] - 最小环 

State: Accepted

From: [NOIO2020]

Algorithm: 贪心 

Author: cyh_toby

Last updated on 20203.12

**********************************/
#include <cstdio>
#include <algorithm>
#include <map>
using namespace std;
#define ll long long
const int N = 200005;
int n, m, p, q;
ll a[N];
map<int, ll> ans;
ll res;
int gcd(int x, int y) {
	if (y == 0)
		return x;
	return gcd(y, x % y);
}
inline void work(int l, int r) {
	if (r==l) {
		res += a[l]*a[l];
		return;
	}
	if (r-l==1) {
		res += a[l]*a[r]*2ll;//注意，根据题目的古怪附加说明，这里要*2 
		return;
	}
	for (int i = r; i >= l; i--) {
		if (i == r) continue;
		if (i == r-1) {
			res += a[i]*a[i+1];
			continue;
		}
		if (i == l) {
			res += a[i] * (a[i+1] + a[i+2]);
			continue;
		}
		res += a[i] * a[i+2];
	}
	return;
}
int main()
{
	scanf("%d%d", &n, &m);
	for (int i = 1; i <= n; i++)
		scanf("%lld", &a[i]);
	sort(a+1, a+1+n);
	while (m--) {
		res = 0ll;//初始化 
		int k;
		scanf("%d", &k);
		p = gcd(n, k);//分成小环的数量
		q = n / p;//小环上点的数量 
		if (ans[p]) {
			printf("%lld\n", ans[p]);
			continue;
		}
		for (int i = 1; i <= p; i++)
			work(n-i*q+1, n-(i-1)*q);
		ans[p] = res;
		printf("%lld\n", res);
	}
	return 0;
}
