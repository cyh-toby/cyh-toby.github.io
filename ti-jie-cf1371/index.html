A. Magical Sticks

题意

原题：CF1371A 【Magical Sticks】

给定 nnn 个长度分别为 111 ~ nnn 的木棍，可以任意合并，求最终最多能有多少个长度相同的木棍。

分析

因为是连续的正整数，不难得知要想使得长度相同的木棍数量最多，一定使最大的和最小的配对，第二大和第二小的配对，以此类推。于是答案就是 ⌊1+n2⌋\left\lfloor\dfrac{1+n}{2}\right\rfloor⌊21+n​⌋。

时间复杂度 O(T)O(T)O(T) 。

源码

#include <bits/stdc++.h>

using namespace std;

int main()
{
	int T;
	scanf("%d", &T);
	while (T--) {
		int n;
		scanf("%d", &n);
		printf("%d\n", (1+n)>>1);
	}
	
	return 0;	
}

B. Magical Calendar

题意

原题：CF1371B 【Magical Calendar】

给定正整数 n , rn\,,\,rn,r 。对于整数 k∈[1,r]k \in \left[1, r\right]k∈[1,r] ，作出一个 kkk 列的网格，取横向连续且长为 nnn 的一段 (其中可以存在换行)，并满足这些选择的格子连通，构成图形。
图形进行平移重合算一种。问共有多少种图形。

如图，对于 连通 ：

分析

  • 对于 r<nr < nr<n ，意味着所有 k<nk < nk<n 。那么构成图形必定至少占两行，并且无论如何一定满足 连通 。所以我们直接枚举图形的第一个格子的位置，这样的每一个位置都一一对应一种图形，数量为 kkk。因此这种情况答案是 ∑k\sum k∑k 即 (1+r)×r2\dfrac{\left(1+r\right)\times r}{2}2(1+r)×r​。

  • 对于 n≤rn \le rn≤r ，显然在 k<nk < nk<n 时仍满足上述分析。对于 n≤kn \le kn≤k 的情况，构成的图形可能只占一行，也可能占两行。但若占两行，容易发现无法满足 连通，因此合法的只能占一行。又由于长度固定为 nnn ，且平移重合的为一种，因此在这种情况下不管 kkk 能有多少取值，总共只有 111 种。所以答案为 [1+(n−1)]×(n−1)2+1\dfrac{\left[1+(n-1)\right]\times \left(n-1\right)}{2} + 12[1+(n−1)]×(n−1)​+1 。

时间复杂度 O(T)O(T)O(T)。

源码

#include <bits/stdc++.h>

using namespace std;
typedef long long ll;

int main()
{
	int T;
	scanf ("%d", &T);
	while (T--) {
		ll n, r;
		scanf("%lld%lld", &n, &r);
		ll ans = 0;
		if (r < n) {
			ans += (1 + r) * r / 2;
		}
		else {
			ans += (1 + n - 1) * (n - 1) / 2;
			ans++;
		}
		
		printf("%lld\n", ans);
	}
	
	return 0;	
}

C. A Cookie for You

题意

原题：CF1731C 【A Cookie for You】

给定正整数 a , b , n , ma\,,\,b\,,\,n\,,\,ma,b,n,m ，有 aaa 个 A 饼干，bbb 个 B 饼干，nnn 个 1 类客人，mmm 个 2 类客人。

  • 1 类：吃一个当前数量较多的那种饼干，若数量相等吃 B；
  • 2 类：吃一个当前数量较少的那种饼干，若数量相等吃 A。

问是否能安排一种客人来的顺序，使得每个客人都在自己的条件下吃到饼干。

分析

我们注意到判定合法的条件是 能吃到饼干，这相当于限制了下界（为 000 ）。用类似于贪心的思路，让 2 类客人先吃是更优的。因为他们会吃数量更少的，这意味着如果有一种饼干数量为 000，还有 2 类客人，就不满足合法了；然而若是数量为 000 时还剩下的是 1 类客人，他们会选择数量更多的另一种（除非另一种也为 000），可见 1 类客人后选一定不会使情况变差。

手动画一画尝试一下发现，如果让 1 类客人穿插到 2 类客人中间，不会使情况更优，因此可以直接让 2 类客人一口气吃完。

所以我们优先选择 aaa 和 bbb 中较小的那个，让它自减 mmm ，判定是否小于 000 判断是否可行。

若可行，我们再只用判断剩下的饼干总数是否不小于 nnn 即可。

时间复杂度 O(T)O(T)O(T)。

源码

#include <bits/stdc++.h>

using namespace std;

typedef long long ll;

int main()
{
	int T;
	scanf("%d", &T);
	while (T--) {
		ll a, b, n, m;
		scanf("%lld%lld%lld%lld", &a, &b, &n, &m);
		
		if (a + b < n + m) {
			puts("No");
			continue;
		}
		
		if (a <= b) a -= m;
		if (b < a) b -= m;
		
		if (a < 0 || b < 0) {
			puts("No");
			continue;
		}
		
		if (a + b >= n) {
			puts("Yes");
			continue;
		}
		else {
			puts("No");
			continue;
		}		
	}
	return 0;	
}

D. Grid-00100

题意

原题：CF1371D 【Grid-00100】

  • 给定正整数 n , kn\,,\,kn,k ，在 n×nn\times nn×n 的矩阵 AAA 内填入 kkk 个 111 ，其余为 000 。记 RiR_iRi​ 为第 iii 行数字和，CjC_jCj​ 为第 jjj 列数字和，f(A)=(max⁡(R)−min⁡(R))2+(max⁡(C)−min⁡(C))2f\left(A\right) = \left(\max(R) - \min(R)\right)^2 + \left(\max(C) - \min(C)\right)^2f(A)=(max(R)−min(R))2+(max(C)−min(C))2

求 min⁡f(A)\min f(A)minf(A)

分析

让 f(A)f(A)f(A) 最小，本质就是尽可能使得每行每列的 111 的个数平均。

注意到 AAA 是个正方形矩阵，我们要尽可能使 111 的个数平均，应该兼顾行列两者，因此按照对角线的方向进行填充。

具体而言，如下：

[1001110001100011]\begin{bmatrix}1&0&0&\color{red}1\\\color{red}1&1&0&0\\0&\color{red}1&1&0\\0&0&\color{red}1&1\end{bmatrix} ⎣⎢⎢⎡​1100​0110​0011​1001​⎦⎥⎥⎤​

[1011110111100111]\begin{bmatrix}1&0&\color{red}1&1\\1&1&0&\color{red}1\\\color{red}1&1&1&0\\0&\color{red}1&1&1\end{bmatrix} ⎣⎢⎢⎡​1110​0111​1011​1101​⎦⎥⎥⎤​

...... ...

我们按照这种法则构造出最优的矩阵 AAA ，然后模拟求解 f(A)f(A)f(A) 。

时间复杂度 O(Tn2)O(T n^2)O(Tn2)。

源码

#include <bits/stdc++.h>

using namespace std;

const int N = 305;

int a[N][N];
int ans = 0;

int main()
{
	int T;
	scanf("%d", &T);
	while (T--) {
		memset(a, 0, sizeof(a));
		ans = 0;
		
		int n, k;
		scanf("%d%d", &n, &k);
	
		int t = k / n;
		
//		printf("t = %d\n", t);
		
		for (int c = 1; c <= t; c++) {
			for (int i = 1; i <= n; i++) {
				if (i+c-1 > n)
					a[i+c-1-n][i] = 1;
				else
					a[i+c-1][i] = 1;
			}
		}
		
		int r = k - t * n;
		
//		printf("r = %d\n", r);
		for (int i = 1; i <= r; i++) {
			if (i + t > n)
				a[i+t-n][i] = 1;
			else
				a[i+t][i] = 1;
		}
		
		int mxr = 0, mnr = n, mxc = 0, mnc = n;
		for (int i = 1; i <= n; i++) {
			int tmp = 0;
			for (int j = 1; j <= n ; j++) {
				tmp += a[i][j];
			}
			mxr = max(mxr, tmp);
			mnr = min(mnr, tmp);
		}
		
		for (int i = 1; i <= n; i++) {
			int tmp = 0;
			for (int j = 1; j <= n; j++) {
			tmp += a[j][i];
			}
			mxc = max(mxc, tmp);
			mnc = min(mnc, tmp);
		}
		
		printf("%d\n", (mxr - mnr) * (mxr - mnr) + (mxc - mnc) * (mxc - mnc));
		for (int i = 1; i <= n; i++) {
			for (int j = 1; j <= n; j++) {
				printf("%d", a[i][j]);
			}
			printf("\n");
		}
		
	}

	return 0;	
}
