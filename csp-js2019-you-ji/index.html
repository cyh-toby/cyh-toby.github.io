第三题——决定等会砸

第四题，又推了一下，发现只用计算每个点到 111 号点的最短路就好，于是开始上BFS。注意，我当时甚至没听说过任何最短路算法，包括 SPFA ，后来才知道这叫求单源最短路——我当时就是真的直接 BFS 推的，加了一些判断条件，出了没有判断是否已经加入队列，完全等于 SPFA 算法！！

先测大样例，貌似过了。异常兴奋。

然后肝 T3T3T3 。

可能我确实不擅长 DP 吧，打了个比树剖还长的 200200200 行暴力，各种分情况讨论，预计 606060 分，结果出来发现只有 151515 。

陷入了分类讨论的泥潭~

还剩 404040 分钟左右，回到 T4T4T4 ，测了一下小样例——没过！！慌了

在纸上画了一下，觉察到和奇偶性有关，但就没深入想了，特判一下三个点的情况。

然后发现其他情况也不对。

慌了更慌了

就是没有想计算奇路径和偶路径。

但在还剩 151515 分钟的时候，牢记早上教训，赶紧检查，不改了。

出来碰到 llx ，他也说 T4T4T4 BFS，然后我说了一堆我都不知道在说什么的分析。

出来听说 T3  T4T3\;T4T3T4都是原题。 mdzz。凉了

后来听讲评，T4T4T4 只是需要分别计算经过长度为偶数的最短路和长度为奇数的最短路就可以了，因为假设一个点到 111 号点的最小长度为 ddd ，如果零件通过这条最短路径后还没到最小级别，就需要继续走。此时，想要从 111 号点走一圈回到 111 号点，必定是再走个偶数长度路径，即一去一回。而 ddd + 偶数，奇偶性不变，所以只用判断 LLL 和 d奇d_{\texttt{奇}}d奇​或d偶d_{\texttt{偶}}d偶​ 是否不大于且同奇偶即可。

T3T3T3 果然是 DPDPDP ，不想说了。。思想就是一天一大早把所有股票卖掉，转化为钱便于计算和统计，因为每一天卖出买入价格相同且不限量。

后来测总分 245245245 ，弱爆了唉。。。

llx 第二题 O(n log n)O(n\,log\,n)O(nlogn) 也过了，总分比我高。
